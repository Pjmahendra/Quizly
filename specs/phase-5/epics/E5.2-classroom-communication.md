# E5.2 — Classroom Communication

## Purpose

Make the classroom the faculty's broadcast channel and record of activity (PRD §5): faculty post
announcements that every enrolled student receives, and the classroom carries a quiz-history
surface so past assessments are visible where the class lives. Code home is `modules/classrooms/`
(announcements are classroom domain), composing `modules/quizzes/` read data through its public
`index.ts` for history (folder-structure §4).

## Business Value

PRD §5 defines announcements and quiz history as core classroom contents, and "students enrolled
in the classroom automatically receive updates" is the classroom's central promise. Announcements
are also a named notification trigger (PRD §14 "Classroom Announcement"), making this epic a
primary producer for the E5.4 platform. Quiz history keeps assessment context inside the classroom
rather than scattered across portals.

## Dependencies

- Phase 1: tenancy and permissions; Phase 2: classrooms and enrollment.
- E5.4 (F5.4.1/F5.4.2) for announcement notification delivery — announcements can ship with the
  in-classroom surface first and gain notifications when the platform lands.
- F5.2.2 reads quiz data owned by Phases 3–4: the surface is built against the quizzes module's
  public interface and renders the shared empty state until quiz data exists — the only E5.1–E5.3
  feature touching quiz phases.
- F5.2.1 attachments reuse E5.1 (the `classrooms/{classroomId}/announcements/` path node,
  folder-structure §10).

## Features

| ID | Feature | Complexity |
|---|---|---|
| F5.2.1 | Classroom announcements | Medium |
| F5.2.2 | Classroom quiz history | Small |

## Acceptance Criteria

- Faculty assigned to a classroom can post, edit, and remove announcements; enrolled students see
  them in the classroom and receive them as notifications once E5.4 is live (PRD §5, §14).
- Announcement lists paginate and are ordered newest-first (guidelines §5).
- Students and faculty see the classroom's quiz history inside the classroom, scoped to that
  classroom and to what the viewer may see (PRD §5; guidelines §8.2), with the shared empty state
  before quiz phases deliver data.
- No classroom-communication logic leaks outside `modules/classrooms/`; quiz data is only read via
  `modules/quizzes` public exports (folder-structure §4, §14).
