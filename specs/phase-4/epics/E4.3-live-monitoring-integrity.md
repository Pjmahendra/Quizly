# E4.3 — Live Monitoring & Integrity

## Purpose

Give faculty live visibility into every student's progress and enforce the quiz integrity policy
in real time. This epic delivers the live monitoring dashboard (participant count, per-student
progress, current question, completion percentage, remaining time — PRD §12), a per-student
drill-in view with inactivity detection and reconnect-event tracking (PRD §4.3 Student
Monitoring), focus-loss detection with configurable warning escalation and auto-termination
(PRD §13), and the per-student activity timeline plus quiz audit log that preserve everything for
later review (PRD §13 Faculty Alerts).

## Business Value

Live monitoring is a core PRD capability (PRD §12) and the reason institutions can trust remote
assessment without heavyweight proctoring: faculty see what is happening as it happens, integrity
violations are handled by policy rather than ad-hoc judgment, and a preserved audit trail makes
outcomes defensible in disputes (PRD §13, §16 Audit Logs). The PRD's scope statement keeps this
realistic: browser activity monitoring only — no webcam, screen recording, or eye tracking.

## Dependencies

- E4.1: authenticated connections, rooms/recovery, and the event catalog — monitoring consumes
  the event stream; integrity events ride the same connection (PRD §13).
- E4.2: session lifecycle — monitoring is only active between `quiz-started` and `quiz-ended`;
  auto-termination reuses the forced-submission path.
- Phase 3 attempts: progress data (answered questions, completion %) derives from attempt state;
  auto-termination is a Phase 3 auto-submit with an integrity marking.
- Phase 2 F2.3.4: the per-quiz integrity policy configuration (enabled/disabled, warning
  threshold) that F4.3.3 enforces (PRD §13 Warning Policy).
- `modules/audit/` for durable audit log entries; Redis `warnings:{attemptId}` for live warning
  counts (folder-structure §9).

## Acceptance Criteria

- The faculty dashboard shows live participant count and, per student: current question, questions
  answered, completion percentage, remaining time, and connection status — updating without page
  refresh (PRD §12).
- Faculty can select any student (`faculty-selected-student`) for a focused live view including
  inactivity indication and reconnect history (PRD §4.3).
- A focus-loss event produces a real-time faculty toast, updates the student's activity timeline,
  writes an audit log entry, and flags the submission for review (PRD §13 Faculty Alerts).
- Warnings escalate per the quiz's configured policy; at the threshold the attempt is
  auto-submitted with remaining questions unanswered and marked "Auto Terminated due to Quiz
  Integrity Policy", with faculty notified immediately (PRD §13).
- Disconnects are never counted as focus losses; the two are visibly distinct on the dashboard
  and in the audit trail (PRD §7 vs. §13).
- The full activity log survives the session and is reviewable after the quiz ends (PRD §13).
