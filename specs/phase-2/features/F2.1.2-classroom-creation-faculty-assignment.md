# F2.1.2 — Classroom creation & faculty assignment

**Epic:** E2.1 — Departments & Classrooms · **Complexity:** Medium

## Description

Faculty can create classrooms (PRD §5 "Faculty can create classrooms"); organization admins can
also create classrooms and assign faculty to them (PRD §4.2 "Create Classrooms", "Assign Faculty").
A classroom is the container that later holds announcements, resources, quiz history (PRD §5 —
those features are later phases) and, in this phase, the enrollment roster (F2.1.3) and quizzes
(E2.3). This feature delivers classroom CRUD, listing, and faculty assignment.

## Dependencies

- Phase 1: tenancy, RBAC (Org Admin, Faculty roles), user management.
- F2.1.1 only if classrooms must reference a department (**Needs clarification** carried from F2.1.1).

## Database Changes (conceptual)

A classroom concept scoped to an organization: name, optional description/subject, active/archived
state, creator, and its assigned faculty. Optionally a department reference, pending F2.1.1's open
question. **Needs clarification:** whether a classroom has exactly one faculty member or several
(PRD §4.2 "Assign Faculty" is silent on cardinality); the concept should not preclude multiple.

## Backend Changes

- New `modules/classrooms/` module (listed in `rules/folder-structure.md` §4) with standard
  anatomy: services (create, update, archive, assign/unassign faculty, list-with-authorization),
  repositories (only Prisma call sites), Zod validators, DTOs, server actions.
- Permission entries via `modules/classrooms/permissions.ts` into `server/permissions/`.

## Frontend Changes

- Faculty-facing classroom creation/management under `app/faculty/` and classroom surfaces under
  `app/classroom/`; org-admin classroom administration under `app/organization/` (shells only).
- Classroom form, list/cards, and faculty-assignment picker in `modules/classrooms/components/`.

## Socket Events

None this phase.

## AI Components

None this phase.

## API Endpoints (purpose-level)

- Create a classroom (faculty or org admin).
- List classrooms visible to the caller (faculty: theirs; org admin: all in org; paginated).
- Get classroom detail (metadata + assigned faculty).
- Update / archive a classroom.
- Assign / unassign faculty (org admin; **Assumption:** the creating faculty member is
  auto-assigned to a classroom they create).

## Permissions

- `classroom.create`: Faculty and Org Admin.
- `classroom.update` / `archive`: assigned faculty and Org Admin.
- `classroom.assign-faculty`: Org Admin only (PRD §4.2).
- Read: assigned faculty, org admin; students see only classrooms they are enrolled in (F2.1.3).

## Validation Rules

- Name required, bounded length; **Assumption:** unique per organization to avoid roster confusion.
- Assigned faculty must be active users with the Faculty role **in the same organization**.
- Tenant key always from server-side context.

## Edge Cases & Failure Scenarios

- Assigning a user who is not faculty, is deactivated, or belongs to another org → rejected.
- Unassigning the last faculty member → **Needs clarification:** allowed (orphaned classroom
  manageable by org admin) or blocked.
- Archiving a classroom with enrolled students or scheduled quizzes → archive hides it from
  students; scheduled-quiz interaction defined in F2.3.2.
- Faculty account deactivated in Phase 1 user management → classroom remains, org admin reassigns.

## Acceptance Criteria

- A faculty member can create a classroom and immediately manage it.
- An org admin can create classrooms and assign/unassign faculty within their organization.
- Faculty see only their classrooms; org admins see all classrooms in their org; no cross-tenant leakage.
- Archived classrooms disappear from student-facing lists but remain auditable.

## Definition of Done

- Meets `rules/definition-of-done.md`; permissions in the central catalog; service→repository→Prisma
  flow only; docs updated per `CLAUDE.md` (feature log + architecture entry for the new module).

## Open Questions

- Faculty cardinality per classroom (one vs. many).
- Department reference: required, optional, or absent (blocked on F2.1.1).
- Can org admins create classrooms with no faculty assigned yet?
