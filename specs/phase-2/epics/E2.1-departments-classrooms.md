# E2.1 — Departments & Classrooms

## Purpose

Establish the academic structure inside an organization: departments created by org admins
(PRD §4.2), classrooms created by faculty (PRD §5) or org admins with faculty assignment
(PRD §4.2), and student membership via enrollment and classroom join (PRD §4.4). This is the
container hierarchy every later phase hangs off — quizzes, resources, announcements, and
analytics all attach to classrooms, and org analytics reports "Department Performance" (PRD §15).

## Business Value

- Mirrors how institutions actually organize teaching, so onboarding maps 1:1 to real structure.
- Unblocks quiz authoring (E2.3): a quiz needs an audience, and the audience is a classroom roster.
- Provides the grouping dimension (department) that organization analytics requires later (PRD §15).

## Dependencies

- **Phase 1:** tenancy (organization scoping), RBAC roles (Super Admin, Org Admin, Faculty,
  Student), user management, and the central permission catalog.
- Internal ordering: F2.1.1 before F2.1.2 only if classrooms reference departments (see the
  **Needs clarification** in F2.1.1); F2.1.2 before F2.1.3 (enrollment needs classrooms).

## Features

| ID | Feature | Complexity |
|---|---|---|
| F2.1.1 | Departments | Small |
| F2.1.2 | Classroom creation & faculty assignment | Medium |
| F2.1.3 | Student enrollment & classroom join | Medium |

## Acceptance Criteria

- An org admin can create, rename, and archive departments within their own organization only.
- A faculty member can create a classroom (PRD §5); an org admin can create classrooms and assign
  faculty to them (PRD §4.2).
- Students can be enrolled by an admin/faculty member and can join a classroom themselves
  (PRD §4.4), with the join mechanism resolved per F2.1.3's **Needs clarification** before build.
- A classroom roster is visible to its faculty; students see the classrooms they belong to.
- All operations are tenant-isolated (no cross-organization visibility) and permission-gated via
  the central catalog; domain code lives in `modules/organizations/` and `modules/classrooms/`
  per `rules/folder-structure.md` §4.
