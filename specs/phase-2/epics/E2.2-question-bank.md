# E2.2 — Question Bank

## Purpose

Give faculty a reusable, searchable bank of questions (PRD §4.3 "Question Bank") covering all six
MVP question types: multiple choice, multiple select, true/false, numerical (objective — auto-graded
per PRD §10), and short answer, long descriptive (subjective — PRD §10 manual evaluation, later
PRD §11 AI evaluation). Every question carries its answer key or reference answer at authoring
time, because PRD §11 step 2 makes faculty-provided expected answers the precondition for Phase 7
AI evaluation.

## Business Value

- Reuse: questions are authored once and composed into many quizzes, the core authoring
  productivity win of PRD §4.3.
- Future-proofing: capturing reference answers now means Phase 7 AI evaluation works on day one
  with no data backfill (PRD §11).
- The type-discriminated question model is the single place the PRD's full type list — including
  future fill-in-blank and file-upload (PRD §4.3) — is anticipated without being built.

## Dependencies

- **Phase 1:** tenancy, RBAC (faculty role), user identity for question ownership.
- Internal ordering: F2.2.1 (core model, CRUD, tagging, search) precedes F2.2.2 and F2.2.3, which
  add the per-type payloads on top of it.
- E2.3 (quiz builder) consumes this epic; it cannot ship before F2.2.1.

## Features

| ID | Feature | Complexity |
|---|---|---|
| F2.2.1 | Question bank core | Large |
| F2.2.2 | Objective question types | Medium |

## Acceptance Criteria

- Faculty can create, edit, tag, search, and archive questions of all six MVP types within their
  bank scope (scope itself is **Needs clarification** in F2.2.1: per-faculty vs. org-shared).
- Objective questions cannot be saved without a complete, auto-gradable answer key (PRD §10).
- Subjective questions capture an expected/reference answer per question (PRD §11 step 2).
- The question model's type discriminator lists fill-in-blank and file-upload as reserved future
  types that cannot be authored (PRD §4.3 "(Future)").
- Editing a banked question never silently changes a quiz that already uses it (reproducibility
  contract shared with F2.3.1/F2.3.3).
- Domain code lives in `modules/questions/` per `rules/folder-structure.md` §4; all queries are
  tenant-scoped and paginated.
