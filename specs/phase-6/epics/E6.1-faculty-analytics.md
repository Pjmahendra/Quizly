# E6.1 — Faculty Analytics

## Purpose

Give faculty the PRD §15 Faculty analytics set — Average Score, Pass Rate, Question Difficulty,
Student Progress, Time Taken, Performance Trends — as dashboards in the Faculty portal, computed
from the graded results data produced in Phase 3 and served from precomputed aggregates
(guidelines §10). This closes the PRD §8 quiz workflow loop: Quiz Ends → Results Generated →
Performance Analytics.

## Business Value

Analytics is what turns a quiz tool into an assessment platform (PRD §2, §3): faculty learn which
questions work, which students are falling behind, and whether outcomes are trending up — insight
they cannot get from raw result lists. It is also the visible payoff of all the data captured in
Phases 3–5, and a core differentiator named in the product scope (PRD §3 "performance analytics").

## Dependencies

- **Phase 3** — attempts, per-question scores, grading outcomes, and timing data are the substrate;
  this epic cannot start before Phase 3 lands. It can run in parallel with Phases 4–5.
- Phase 1 RBAC and tenant scoping (capabilities from the central catalog; org context from trusted
  server context — guidelines §1.1, §9).
- The new `modules/analytics/` module (folder-structure §4) with its snapshot/rollup services, and
  shared chart primitives in `src/components/charts/` (folder-structure §6).
- **Needs clarification (blocking F6.1.1):** the formulas for Pass Rate and Question Difficulty —
  PRD §15 names the metrics without defining them.

## Acceptance Criteria

- All six PRD §15 Faculty metrics render in the Faculty portal for quizzes the faculty member
  owns — and only those quizzes, within their own organization (guidelines §1.1).
- Dashboard reads come from precomputed per-quiz and per-student aggregates with visible
  freshness — no unbounded scan of attempts on the request path (guidelines §2, §10).
- Quizzes with no attempts, or with grading still pending (manual or AI), render honest
  empty/pending states instead of misleading zeros (guidelines §13).
- Pass Rate and Question Difficulty ship only after their formulas are clarified and recorded in
  the feature spec (see F6.1.1 Open Questions).
- Both features pass `rules/definition-of-done.md`.
