# F0.1.1 — App scaffold & strict tooling baseline

**Epic:** E0.1 — Project Scaffold & Tooling · **Complexity:** Medium

## Description

Bootstrap the Next.js 15 application (TypeScript, App Router, Tailwind CSS per PRD §17) with the
exact `src/` layout of folder-structure §2: `app/`, `modules/`, `server/`, `components/`, `hooks/`,
`providers/`, `lib/`, `config/`, `types/`, `styles/`. Seam directories are created with purpose
statements but no domain code (PRD §18 phases begin later). Establish the strict tooling baseline:
strictest TypeScript with no per-file opt-outs (guidelines §4.1), lint and format at zero warnings,
and a pre-commit gate running format/lint/type checks that blocks rather than warns (guidelines §20).

## Validation Rules

- TypeScript strict mode enabled globally; escape hatches (`any`, ignore directives) require a justifying comment (guidelines §4.1).
- Lint passes at zero warnings; formatting is enforced, not advisory.
- Pre-commit gate cannot be bypassed as a matter of workflow (guidelines §20).
- Import discipline per folder-structure §14: cross-module imports only via `index.ts`, never deep paths (guidelines §4.3).

## Edge Cases & Failure Scenarios

- Fresh clone on a machine without the documented Node.js version — setup docs must state the supported version so failure is explicit, not mysterious.
- Generated artifacts (build output, dependency folders, generated Prisma client) must be excluded from lint/type gates and version control.
- Empty seam directories must survive in git (they carry purpose statements, so they are non-empty).
- A contributor bypassing hooks locally — CI-side enforcement is the backstop (guidelines §20 "never bypass the gate").

## Acceptance Criteria

- `npm`-equivalent install + dev-server boot succeeds from a fresh clone and renders the placeholder page.
- The `src/` tree matches folder-structure §2 exactly; route files contain no business logic (folder-structure §3).
- Type-check, lint, and format all pass at zero warnings; the pre-commit gate demonstrably blocks a violating commit.
- No auth, socket, AI, queue, or domain-module code exists.

## Definition of Done

Passes `rules/definition-of-done.md` (esp. §17 Quality Gates) and the guidelines §4.1/§20 gates at zero warnings; plus: structure verified against folder-structure §2, and `docs/architecture.md` + a `docs/logs/` entry created per `CLAUDE.md`.
