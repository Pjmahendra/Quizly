# F0.1.3 — Local service connection wiring

**Epic:** E0.1 — Project Scaffold & Tooling · **Complexity:** Small

## Description

Provide typed connection configuration for the three local backing services — PostgreSQL, Redis,
and Azurite (PRD §17; PRD §9 storage strategy) — per the `CLAUDE.md` local-development rules: all
three run in local Docker containers, never as host installs, and local connection strings target
`localhost` only, never cloud/managed instances. Connection shapes live in `config/database.ts`,
`config/redis.ts`, and `config/storage.ts` (folder-structure §7), sourced exclusively from the
validated env (F0.1.2). A repo-root `docker-compose.yml` (folder-structure §1; `CLAUDE.md`)
declares the three services for local development. Configuration only — no cloud provisioning and
no deployment concerns.

## Validation Rules

- Connection values are validated for shape at boot via `lib/env.ts` (F0.1.2); malformed values fail fast.
- Local-development values must point at the Dockerized `localhost` services (`CLAUDE.md`); pointing local dev at a cloud instance is a violation.
- No real credentials are committed; Azurite's published development credentials are explicitly documented as non-secrets to avoid confusion with real secret handling (guidelines §8.6).

## Edge Cases & Failure Scenarios

- A service container not running — the app should surface an actionable connectivity failure (and F0.2.5 readiness reflects it) rather than a cryptic driver error.
- Host port collisions with existing local services — ports are configurable through env, defaults documented.
- Containers up but not yet accepting connections (cold start) — readiness probing (F0.2.5) handles this; boot must not permanently cache a failed state.

## Acceptance Criteria

- With the three containers running, the application boots and each config module resolves a valid typed connection shape.
- All connection configuration flows env → `lib/env.ts` → `config/` module; no connection string is hardcoded anywhere.
- The repo-root `docker-compose.yml` brings up PostgreSQL, Redis, and Azurite for local dev per `CLAUDE.md`.
