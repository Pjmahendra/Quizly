# F0.3.2 — Shared UI primitive library seed

**Epic:** E0.3 — Design System Foundation · **Complexity:** Large

## Description

Seed the shared, feature-agnostic UI library in its folder-structure §6 homes: `components/ui`
(button, input, dialog), `components/tables` (shared table, pagination), `components/forms` (form
field seeds), and `components/feedback` (toast, empty state, skeleton). These are the abstractions
guidelines §11 requires to exist exactly once ("tables, pagination, search/filter, empty states,
loading skeletons, forms, and dialogs come from shared components — never hand-rolled per screen")
and guidelines §13 mandates for all product surfaces. Every primitive styles itself exclusively
through F0.3.1 tokens, expresses the full interaction-state set (guidelines §12), meets WCAG 2.1 AA
(guidelines §14), and renders correctly in both themes. Only feature-agnostic UI lives here —
anything portal- or domain-specific stays in its future module (folder-structure §6).

## Validation Rules

- No feature-, portal-, or domain-specific logic in any shared component (folder-structure §6).
- Tokens only — no raw style values (guidelines §12; F0.3.1).
- Every interactive primitive expresses rest, hover, focus, active, selected, disabled, and loading states via the shared state-layer system (guidelines §12).
- Keyboard operability, visible focus, programmatic labels, and ≈44px touch targets (guidelines §14).
- State never conveyed by color alone (guidelines §14).

## Edge Cases & Failure Scenarios

- Dialog: focus trap, escape/overlay dismissal, and focus return to the trigger; destructive-confirm variant supported (guidelines §13).
- Table with zero rows renders the shared empty state, never a blank grid; wide tables scroll within their container, not the page (guidelines §12, §13).
- Toast stacking and overflow — bounded, dismissible, honoring reduced motion.
- Skeletons under reduced-motion — non-animated equivalents (guidelines §14).
- Pagination at boundary pages (first/last/single page) — controls disable coherently and the count stays truthful (definition-of-done §7).

## Acceptance Criteria

- All eight primitives exist in their folder-structure §6 homes and are importable by future modules.
- The preview surface shows every primitive in both themes with all interaction states, and passes a keyboard-only walkthrough (guidelines §14).
- No shared component references any domain concept.
- Tabular/paginated patterns match the guidelines §13 layout rules.

## Definition of Done

Passes `rules/definition-of-done.md` (esp. §2 Data States, §7 Lists/Tables, §8 Feedback, §9 Forms, §12 Accessibility Experience) and the guidelines §4.1/§20 gates at zero warnings; plus: both-theme rendering verified, and docs updated per `CLAUDE.md`.
