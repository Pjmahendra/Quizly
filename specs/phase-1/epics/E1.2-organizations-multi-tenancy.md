# E1.2 — Organizations & Multi-Tenancy Core

## Purpose

Make organizations first-class isolated tenants (PRD §4.1–4.2) and build the trusted server-side
tenant-scoping context that every scoped query in every later module reads (guidelines §1.1,
§8.2). Includes Super Admin organization CRUD and org admin assignment/onboarding.

## Business Value

- Quisly sells to institutions (PRD §3); institutional isolation is the product's baseline trust
  promise. A cross-tenant leak is the worst possible failure.
- Every Phase 1–3 domain (classrooms, quizzes, resources, analytics) is org-scoped; getting the
  scoping seam right once means each later module inherits isolation instead of re-implementing
  it (guidelines §1.1 — defense at a single enforcement point per domain).
- Super Admin org lifecycle (PRD §4.1) is the commercial onboarding path: no organization, no
  customers.

## Dependencies

- E1.1 — an authenticated principal must exist before it can be resolved to a tenant scope.
- F1.3.1 permission catalog for guarding org CRUD (can be developed in parallel; org endpoints
  gate on it before release).
- F1.4.4 audit log platform for recording org lifecycle mutations (guidelines §8.8).

## Acceptance Criteria

- An organization exists as an entity with a lifecycle managed exclusively by Super Admin
  (PRD §4.1).
- Every scoped read/write in the codebase derives the organization key from the server-side
  scoping context — never from a client-supplied identifier — enforced at the repository seam
  (guidelines §1.1, §8.2; folder-structure §8).
- Resource IDs in any request are verified to belong to the caller's tenant scope before acting
  (IDOR/BOLA defense, guidelines §8.2).
- A Super Admin can create, edit, and delete organizations; deletion is a guarded, audited,
  deliberate act (PRD §4.1; guidelines §8.8).
- An organization admin can be assigned to an organization and can complete initial org setup
  after first sign-in (PRD §4.1–4.2).
- Cross-tenant access attempts fail closed and are indistinguishable from "not found" to the
  caller.
- All features pass `rules/definition-of-done.md`.
