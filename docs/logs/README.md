# Feature implementation logs

Every implemented feature gets one log file in this folder, per CLAUDE.md's primary project rule.

**Naming:** `YYYY-MM-DD-<feature-name>.md` (date the feature was completed, kebab-case name).

**Each entry must cover:**

- **What** was implemented
- **Why** it was implemented
- **How** it works
- Limitations
- Dependencies
- Configuration requirements
- Future considerations (if applicable)

If an implementation changes an already-logged feature's behavior, update that feature's existing
log entry rather than creating a duplicate. Architectural impact additionally goes in
[`docs/architecture.md`](../architecture.md).
