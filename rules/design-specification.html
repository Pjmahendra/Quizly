Emphasis Ladder
Emphasis sets how much a color contrasts with the default surface: subtlest → subtler → subtle → (default) → bold → bolder → boldest. Bold draws primary attention (bold lozenges, primary buttons); subtle/subtlest de-emphasize secondary UI. Interaction states append last: color.background.brand.bold.hovered, .pressed, plus selected, focused, disabled.
Accents — the Swap Test
Accent hues (gray, red, orange, yellow, lime, green, teal, blue, purple, magenta) carry no meaning: "You should be able to exchange one accent color for another, and the experience would remain unchanged."

✗ STRICT: never use an accent where the color has semantic meaning — no red accent for errors (use danger), no yellow accent for caution (use warning).
Dark Mode
Both themes ship from the same tokens — toggle with the theme button in the sidebar. Neutrals map 1:1 to the DarkNeutral ramp; saturated ramps mirror by symmetry (light 700 → dark 400); dark alphas are light-tinted. Elevation in dark mode uses lighter surfaces at higher levels (see Elevation section). color-scheme is set per theme so native controls follow.
STRICT — token-only color. "Don't use a token just because the colors appear to match. This can break the experience in other themes." Raw hex values are forbidden in components; enforce with @atlaskit/design-system/ensure-design-token-usage (ESLint) or the Stylelint equivalent.

Strict Heading Rules
"Heading levels (<h1>–<h6>) should be used in a descending sequence. Only use one h1 per page (usually the page title) and don't skip a level."

Size selection is role-based: XXL/XL → brand & marketing · XL/L → app page titles · M → large components (modals) · S/XS → small components (flags) · XXS → fine print, sparingly, matched with body.small.
Strict Body & Code Rules
Body text follows headings as descriptions and messages. body.large for long-form reading · body (14px) is the component default · body.small sparingly for fine print — 12px is the smallest sanctioned size.

Elevation — ADS
"Elevations are layered surfaces that form the foundation of UI." An elevation is always a pair: a surface color token + its matching shadow token. STRICT: never mix surface and shadow tokens from different levels. In dark mode, shadows are harder to see, so elevation additionally relies on lighter surfaces at higher levels.
1 · surface.sunken
Lowest perceived level — a backdrop behind content (board backgrounds). No shadow. #F8F8F8 / #18191A

✗ Never place sunken surfaces on raised or overlay levels.
2 · surface (default)
Baseline app background. No shadow. #FFFFFF / #1F1F21

Hovered/pressed variants exist for every level except sunken.
3 · surface.raised + shadow.raised
Raised elements such as cards. #FFFFFF / #242528 with elevation.shadow.raised.
4 · surface.overlay + shadow.overlay
UI floating above other UI: modals, dropdowns, popups. #FFFFFF / #2B2C2F with elevation.shadow.overlay (in dark mode the overlay shadow includes a faint light ring for edge definition).

Border & Radius — ADS
Borders "define boundaries, separate components, and add visual emphasis"; radius tokens "standardize corner roundness". STRICT rule of thumb: width controls visibility, color communicates purpose. The radius scale is role-based and grows with element size.
Border Width Tokens
border.width — 1px · default for all component borders and dividers
border.width.selected — 2px · selected elements (active tab, chosen item) — pair with color.border.selected
border.width.focused — 2px · focus ring — pair ONLY with color.border.focused
There is no border.width.outline token. On subtle backgrounds, pair with an accent border to meet the 3:1 contrast requirement.

⑤ Alignment & Proximity Rules
Consistent alignment creates visual order. Related elements must be grouped closer together than unrelated ones. All alignment derives from the 4px base unit.
Left-Align Everything
All text, labels, and UI elements align left in LTR contexts. Centre-align only for hero headlines and empty states.
4px Base Grid
Every size, spacing, and position is a multiple of 4px. No half-pixels, no arbitrary values. Snap to the grid.
Proximity Grouping
Related items: 8px gap. Component-to-component: 16–24px. Section-to-section: 40–64px.
Optical Alignment
Icons and text align to their optical centre, not their bounding box. Adjust by 1–2px where needed.

The design system uses ripple effects on press to confirm interaction. For Quizly: use a CSS pseudo-element ripple on buttons and list items. Ripple color = content color at 12% opacity. Duration = 250ms ease-out.

Motion Guidelines
Our motion system uses Emphasized, Standard, and Decelerate easing curves. Duration is context-driven: quick for small elements, slower for large containers. Always respect prefers-reduced-motion.
Easing Curves
Emphasized: cubic-bezier(0.2, 0, 0, 1)
Standard: cubic-bezier(0.2, 0, 0, 1)
Decelerate: cubic-bezier(0, 0, 0, 1)
Accelerate: cubic-bezier(0.3, 0, 1, 1)
Spring: cubic-bezier(0.34, 1.56, 0.64, 1)
Decelerate = enter. Accelerate = exit. Emphasized = transform/morph.

Icon Design Rules — ADS Standard
Rule 1 — Size
16×16px — medium (default): use in most cases; balances with body text and app density
12×12px — small (sparingly): chevrons (always 12px in buttons/dropdowns), field validation messages, tags/badges/statuses, secondary supporting roles

Larger decorative needs → an icon tile (icon on a colored background tile), never a raw scaled-up icon. No sizes between stops.
Rule 2 — Stroke Width
Always use weight="regular" — the single standard weight for all UI. weight="fill" is permitted only as an active/selected indicator (e.g. filled nav item). Thin, light, bold, and duotone are forbidden in product UI. Never mix weights within the same surface, and never restyle glyph geometry — ADS construction rules (sharp internal corners, no 3D or diagonal perspective) are satisfied by the Phosphor set itself.
Rule 3 — Color Inheritance
Color comes from icon tokens only: color.icon, .subtle, .subtlest, .inverse, or the matching text token — never hardcoded hex. Icons have no hovered/pressed tokens: indicate state with background.neutral.subtle.hovered/.pressed behind the icon. Spacing around icons comes from the space scale (Box padding), never hardcoded margins.
Rule 4 — Semantic Meaning
One icon, one meaning. Use widely recognized symbols and established visual metaphors — never culture- or language-dependent imagery. Reuse existing icons before creating new ones (e.g. CheckCircleIcon = success only, WarningIcon = warning only). Keep icons simple enough to read at 16px. Document usage decisions in Storybook.
Rule 5 — Alignment & Gap
Always vertically center-align icons with accompanying text. Gap between icon and label: 8px (sp-2). Never place icons after text labels for primary actions. Leading icons only — trailing arrows are the only exception.
Rule 6 — Accessibility
Decorative icons: aria-hidden="true". Standalone icon buttons: aria-label="Action name". Never rely on icon alone to convey state. Pair icon with visible label text or tooltip in all interactive contexts.

Tailwind CSS — Strict Best Practices
✓
Semantic utilities only, mapped to tokens in @theme: bg-primary, text-muted-foreground, border-border, rounded-md
✓
size-4 over h-4 w-4; gap-* over child margins; spacing utilities only from the ADS scale
✓
Conditional classes via cn() (clsx + tailwind-merge) — never string concatenation
✓
Class order enforced by prettier-plugin-tailwindcss; states via hover: focus-visible: disabled: aria-* variants
✓
Mobile-first responsive prefixes; breakpoints follow the ADS grid (sm 768 · md 1024 · lg 1440 · xl 1768)
!
Never arbitrary values (w-[13px], text-[#1868DB]) or raw palette utilities (bg-blue-600) — if a value isn't a token, it doesn't ship
!
Never dark: color overrides in components — dark mode is resolved entirely by the token layer
!
Never @apply in component code and never inline style= for anything a utility covers
